## Ответы на вопросы
## 1. Что делает fetch?
Метод fetch отправляет HTTP-запросы из браузера и возвращает Promise. Он работает асинхронно, не блокирует интерфейс и используется для загрузки данных, например цитат или изображений.
## 2. Зачем нужны async/await?
async/await упрощает работу с асинхронным кодом. Код читается линейно, без цепочек then(), а ошибки обрабатываются через try...catch.
## 3. Как работает localStorage?
localStorage — это постоянное хранилище браузера формата «ключ-значение». Данные сохраняются между перезагрузками. В работе используется для сохранения выбранной темы.
## 4. Где помог ИИ, а где пришлось разбираться вручную?
ИИ помог: сгенерировать основу кода, настроить API, исправить ошибки и оптимизировать логику загрузки.
Вручную пришлось разбираться: с каруселью, точками переключения, автопрокруткой, обработкой статуса «Загрузка…» и применением темы через localStorage.

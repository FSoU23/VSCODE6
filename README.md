Ответы на вопросы
1. Что делает fetch?

Метод fetch используется для отправки HTTP-запросов из браузера.
Он:

работает асинхронно и не блокирует интерфейс;

возвращает Promise;

позволяет получать данные в формате JSON, текст, изображения и т.д.;

используется для интерактивных элементов сайта, которые должны загружать данные без перезагрузки страницы.

В моей работе fetch применён для загрузки цитат и изображений с внешних API.

2. Зачем нужны async/await?

async/await — синтаксическое упрощение для работы с Promise.

Использование async/await даёт следующие преимущества:

код становится чище и легче читается, чем цепочки .then();

не возникает “callback hell”;

можно обрабатывать ошибки через обычный try…catch;

исполнение кода визуально выглядит как синхронное, хотя фактически остаётся асинхронным.

В лабораторной работе async/await использовались для загрузки цитат и изображений, чтобы корректно дождаться завершения всех запросов.

3. Как работает localStorage?

localStorage — это встроенное хранилище браузера формата “ключ-значение”.
Особенности:

данные хранятся постоянно, пока пользователь вручную не очистит;

при перезагрузке страницы значения сохраняются;

хранит только строки, поэтому объекты нужно сериализовать через JSON.stringify.

В работе localStorage использован для сохранения выбранной темы (светлая/тёмная), чтобы она автоматически применялась при следующем визите на страницу.

4. Где помог ИИ, а где пришлось разбираться вручную?

ИИ помог:

реализовать асинхронную загрузку цитат и галереи;

оптимизировать функции рендера отзывов и изображений;

исправить проблемы с обновлением цитат;

подобрать корректный API после отказа исходного.

Пришлось разбираться вручную:

с логикой карусели (переключение слайдов, индикаторы, autoscroll);

с обработкой ошибок и правильным выводом «Загрузка...»;

с адаптацией API под структуру моей верстки;

с localStorage и корректным применением темы при загрузке страницы
